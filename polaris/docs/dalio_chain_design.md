# 达利欧因果链设计

> 核心问题：当前周期环境下，应该持有什么类型的资产？主动押注的风险收益结构是什么？

## 与巴菲特链的根本区别

| | 巴菲特 | 达利欧 |
|---|---|---|
| **作用对象** | 单一公司 | 先资产类型，再落公司 |
| **时间依赖** | 几乎无 | 完全依赖周期位置 |
| **数据刷新** | 季报 | 宏观月度/周度 |
| **持有期** | 永久 | 跟周期走 |
| **利润来源** | 买到好生意的内在价值增长 | 多个低相关主动押注 × 分散化 |
| **结论有效期** | 年级 | 月级，环境变则结论变 |

## 三层盈利结构

```
顶层: 尾部保护 (hedge_specs)
  — 截断"判断错了"的最大损失
  — 不产生收益，是保险成本

中层: 主动押注 (active_tilts)
  — 利润来源
  — 基于周期判断的方向性偏移
  — 多个低相关押注，每个 55-60% 胜率
  — 分散化本身提升夏普比

底层: 风险平价 (risk_parity_baseline)
  — 全天候底仓，不依赖任何判断
  — 四象限都有资产，任何环境净值不崩
  — 赚的是 beta 风险溢价，不是 alpha
```

## 链结构

### 线 1: 周期定位 → 类型选择（串行）

#### ① 宏观象限定位 (quadrant.py)

纯宏观，与个股无关。

输入: 宏观指标（利率、GDP、CPI、信贷增速、央行政策）

输出: `CycleRegime`
- 增长/通胀四象限
- 短期债务周期阶段: 扩张初期 / 扩张末期 / 紧缩初期 / 紧缩末期
- 长期债务周期阶段: 加杠杆 / 去杠杆(丑陋) / 去杠杆(漂亮)
- 定位置信度

四象限资产偏好:
```
                  增长 ↑                  增长 ↓
通胀 ↑    股票/大宗商品            大宗商品/通胀挂钩债
通胀 ↓    股票/名义债券            名义债券/现金
```

**判断依据**:
- GDP 增速 vs 趋势 → 增长方向
- CPI / PCE vs 目标 + 趋势 → 通胀方向
- 信贷增速 / 利率周期位置 → 短期债务周期阶段
- 总债务/GDP + 趋势 → 长期债务周期阶段

#### ② 传导暴露分析 (transmission.py)

宏观力量如何传导到具体资产类型。

四个传导通道:
- **利率传导**: 久期、浮动利率占比、再融资时间表
- **需求传导**: 收入周期性（必需品 vs 可选消费 vs 资本品）
- **大宗商品传导**: 成本结构中原材料占比
- **汇率传导**: 收入/成本币种错配

输出: 每个通道的暴露度 + 当前象限下是利好还是利空

#### ③ 主动押注生成 (tilts.py)

基于 ①② 生成具体的超配/低配建议。

每个 Tilt 包含:
- 资产类型 + 方向 + 幅度
- 因果逻辑（thesis）
- 置信度
- 失效触发条件（什么信号出现时撤回押注）

**关键**: 不是一个大判断，而是多个低相关的小判断。
每个押注独立，互不依赖。

#### ④ 风险平价基线 (risk_parity.py)

不依赖任何判断的被动底仓。

- 等风险贡献: 每个资产类别贡献相等的组合风险
- 需要: 各资产类别的波动率 + 相关性矩阵
- 这层独立于 ①②③，始终存在

### 线 2: 认知盲区（并行，始终运行）

#### A. 尾部风险 (tail_risk.py)

"What you don't know is more important than what you know."

检测:
- 交叉违约条款 → 连锁加速到期
- 表外负债 / 衍生品 → 隐藏杠杆
- 客户/供应商集中度 → 单点崩溃
- 与系统性风险的相关性（紧缩期 beta > 1.5 = 危险）

输出: 尾部风险标记（不直接否决，但影响对冲规格）

#### B. 政策路径 (policy_path.py)

达利欧的四个政策工具:
1. 降息
2. 量化宽松 (QE)
3. 财政刺激
4. 货币贬值

评估:
- 当前政策空间: 利率还能降多少？赤字率还有多少余量？
- 政策响应概率: 如果经济恶化，政策制定者会用哪个工具？
- "漂亮去杠杆"需要四个工具配合 → 目前是否具备条件？

输出: 政策路径对各资产类型的影响方向

### 交叉判断 → 对冲规格

线 1 的 active_tilts + 线 2 的尾部风险 + 政策路径 → 生成 hedge_specs

每个 HedgeSpec:
- 保护什么反面情景
- 可接受的最大亏损
- 当前配置在该情景下最大的裸露敞口

**Polaris 只输出 hedge_specs（"对冲什么"），Axion 决定"用什么工具、多大仓位"。**

## 第 ③ 步: 载体落地（暂缓）

在选定类型中筛选具体公司作为载体。

选股标准（与巴菲特完全不同）:
- **敞口纯度**: 收入/利润结构是否集中在目标类型
- **流动性**: 市值、交易量足够大
- **周期存活性**: 现有 11 条规则条件化于 CycleRegime
- **相关性贡献**: 与已选载体的低相关性

暂缓原因: 需要市值、日均成交量、相关性矩阵等数据，当前管线不支持。

## Polaris → Axion 接口

```python
@dataclass
class DalioResult:
    regime: CycleRegime | None
    risk_parity_baseline: dict[str, float]   # Axion 用: 被动底仓
    active_tilts: list[Tilt]                 # Axion 用: 主动偏移执行
    hedge_specs: list[HedgeSpec]             # Axion 用: 对冲工具选择
```

Axion 收到后:
1. 按 risk_parity_baseline 配底仓
2. 按 active_tilts 叠加方向性偏移
3. 按 hedge_specs 选工具截断尾部
4. 监控每个 tilt 的 decay_trigger，触发时自动撤回

## 实现优先级

1. **P0**: CycleRegime 定位 + 四象限映射（纯宏观，可独立跑）
2. **P1**: active_tilts 生成 + hedge_specs（核心价值）
3. **P2**: risk_parity_baseline（需价格数据 + 波动率）
4. **P3**: 载体落地（需完整公司级数据）
